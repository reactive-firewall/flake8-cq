SIM211: Use 'not a' instead of 'False if a else True'

Use the pythonic 'falsy' form; It reduces cognitive complexity slightly.

Complex Form:

False if a else True


Simplified From:

not a


Rationale:
Both notations are semantically equivalent in Python 3, but the second one is shorter.
